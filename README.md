## How to Report a Bug

Before reporting an issue:
1. Check whether the issue has already been reported
2. Update the application to the latest version

When submitting a bug report, please include:
- App version
- Device model and Android version
- Steps to reproduce the issue
- Expected behavior
- Screenshots or screen recordings (if available)

Use the **Bug Report** issue template when creating a report.

---

## Request a Feature

Have an idea that could make billing easier or more efficient?

When requesting a feature:
- Explain the problem you are facing
- Describe how the feature would help your business
- Mention your business type (Retail, Medical, Construction, etc.)

Use the **Feature Request** issue template for suggestions.

---

## Issue Labels

- `bug` – Something is not working correctly
- `feature-request` – New feature suggestion
- `enhancement` – Improvement to an existing feature
- `ui/ux` – Design or usability feedback
- `high-priority` – Critical issues affecting billing
- `needs-info` – Additional details required
